<objective>
Add compiler-enforced branded types (`SafeHyperlinkUrl`, `SafeHtml`) and smart constructors
(`toSafeHyperlinkUrl`, `toSafeHtml`) to `openclerk-core` -- the shared logic package consumed by
`openclerk-word` (and, per its README, Google Docs/LibreOffice integrations) -- so that only a
validated/escaped value can ever satisfy these types, per D-01/D-02. Bump `openclerk-core` to a
new version (0.3.0) and stop at a human checkpoint before the actual npm publish, per D-03 --
matching the PR #33 merge-checkpoint pattern from Phase 1.

Purpose: ESCAPE-01 requires branded types that type-constrain Office.js insertion helpers. Per
D-01 these types belong in `openclerk-core` (the validation source of truth), not a local
`openclerk-word` file, so every consumer gets the compile-time guarantee, not just Word.
Output: `openclerk-core` v0.3.0, published to the public npm registry after human confirmation,
exporting `SafeHyperlinkUrl`, `SafeHtml`, `toSafeHyperlinkUrl`, `toSafeHtml`.
</objective>
